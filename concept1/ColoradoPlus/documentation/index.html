02. Included Files

Table of Contents

There are a total of eleven HTML files inlcuded in the download package. With it being a "one-page" design, your main focus would be on the "index.html" file. Although there are blog pages in the template, they are not functional and need to be integrated into a CMS. Here's a quick overview of each file;

  • index.html - The main file that contains all of the single page content and functionality.
  • template-blog-overview.html - Displays your blog's front page in a masonry style layout.
  • template-blog-single-full-width.html - A single full-width blog entry page with no sidebar.
  • template-blog-single-left-sidebar.html - A single blog entry with the sidebar on the left. This template uses the pull/push CSS classes to keep the main content first in the source.
  • template-blog-single-narrow.html - Similar to the full-width layout but with a narrow column for the body content.
  • template-blog-single-right-sidebar.html - A single typical blog entry with the sidebar on the right.
  • template-menu-one-column.html - Displays a menu layout in a single, narrow column.
  • template-menu-two-columns.html - Displays the menu in a two column layout, similar to the one on "index.html".
  • template-menu-three-columns.html - Displays the menu in a grid-style three column layout.
  • template-recipe-single.html - A single page entry for a recipe with meta information in the left column.
  • template-recipe-single-swap.html - Another single page recipe entry with the meta information on the right (uses pull/push technique).

03. HTML Structure

Table of Contents

The HTML code used in this template complies with all of the W3C guidelines and have all been validated. Editing each of the sections within the main "index.html" file is relatively simple. Here's what a typical section will look like;

<section id="hungry-about-us" class="section-container">
	<header class="section-heading">
		<h2 class="section-heading-title">About Us</h2>
		<div class="section-heading-subtitle-container tilt-left">
			<h4 class="section-heading-subtitle">Whats our Story?</h4>
		</div>
	</header>
	<div class="grid-container">

		...section content...

	</div>
</section>

As you can see it starts of with a section heading, sub-heading and then the main content. Altering your content is as easy as replacing the content with your own to suit your needs.

About the Grid System

Hungry uses a flexible and robust CSS grid system to layout certain content into columns. It's used for almost every element that is split up to two columns or more. Here's a brief example;

<div class="grid-container">
	<div class="grid-50 tablet-grid-50 mobile-grid-100">
	
		...First Column Content...
	
	</div>
	
	<div class="grid-50 tablet-grid-50 mobile-grid-100">
	
		...Second Column Content...
	
	</div>
</div>

This example shows a simple two column grid layout starting with a grid-container, which is needed to contain the columns. Within it, there are two <div>s with the class name grid-50. Think of the number value as a percentage. In this case, each column is 50 percent wide so it will be split into two equal halves. You can also control the layout in different screen resolutions for smaller devices with the tablet and mobile prefixes. In the above example, desktop and tablets will display the content over two columns, while mobiles will display it in a single column.

For more information on this grid system, please visit the documentation page over at their website at unsemanticgrid.com

05. Configuring the Page Header

Table of Contents

At the very top of the single-page template (index.html) is where the full screen slider, animated text and social icons reside. This chapter will go over how to configure these.

The Fullscreen Slider

The slider uses the Cycle 2 Slider jQuery plugin. all of the options for this are found directly in the HTML file itself using data attributes;

<div class="cycle-slideshow"
	 data-cycle-swipe="true"
	 data-cycle-swipe-fx="fade"
	 data-cycle-fx="fade"
	 data-cycle-speed="1200"
	 data-cycle-timeout="16000">
	
	<img src="images/demo/slide-image-01.jpg" alt="Slide Image 01" />
	<img src="images/demo/slide-image-02.jpg" alt="Slide Image 02" />
	...
</div>

The above code snippet shows all of the available options for the slider. data-cycle-fx can be set to either "fade" or "slideHorz". The speed and timeout options can be both set in milliseconds. You can have as many images as you want by inserting the images directly after one another, like in the example above. Just keep in mind that the more images you have, the longer your site will take to load!

The optimal image size for this slider is 1920 x 1000 pixels. This ensures that they look good on wide screen displays.

Animated Slogans

The template uses the Textillate jQuery plugin to animate each of the slogans. Editing these is as simple as editing an unordred list. Look for the following code which is found directly after the slideshow code above;

<div class="tlt">
	<ul class="header-texts">
		<li>Dine with us!</li>
		<li>Try the Wine!</li>
		...
	</ul>
</div>

As with the images, you can have as many slogans as you want! There are also some options to configure if you need to change things like the animation and other behaviours. These can be found in "jquery-custom.js" which is located in the "js" folder.

Social Icons & Tooltips

The social icons are also laid out in a list and have Tooltips attached to them. They also use the FontAwesome icons to display the social media icons;

<div class="single-page-social-icons">
	<ul class="single-page-social-icons-list">
		<li><a href="#" class="header-social-icon-tooltip" title="Follow us on Facebook!"><i class="fa fa-facebook"></i></a></li>
		<li><a href="#" class="header-social-icon-tooltip" title="Find us in Twitter!"><i class="fa fa-twitter"></i></a></li>
		...
	</ul>
</div>

To change the tooltip text, you simply need to alter the text in the "title" attribute. Again, you can have as many as you wish but it is recommended that you only have around five or six so that there are no layout issues.

06. Changing Colour Schemes

Table of Contents

You may notice a light brown accent colour that gets used throughout the design, such as in certain headings and hovered elements. To change this colour, all you need to do is go to the very bottom of the main stylesheet (style.css) and find chapter 11 - ACCENT COLOUR. This chapter targets all the CSS classes and ID's that use the accent colour in order to keep it in one place.

The default colour is #ccad52;. If you have a search and replace function in your text editor, do a search for this string of text and replace it with a colour of your choice!

There is also a divider image that uses this colour, which will have to be changed manually. The PSD file for this is included within the download package. Simply alter what you need with Photoshop (or similar software), and save over the original PNG file which is found in the "images/assets" folder.

07. Changing Fonts

Table of Contents

There is also a section in the main stylesheet (style.css) that controls the fonts used in the template. If you wish to change the fonts, you will need to do two things. The first thing is to edit the URL for the link at the top of each HTML file. It will look like this:

<link href="http://fonts.googleapis.com/css?family=Ubuntu:400,700%7CDroid+Serif:400,400italic,700,700italic%7CAguafina+Script" rel="stylesheet" type="text/css">

This links to Google's font library to get the fonts, so to change fonts, you will need to alter this URL. Selecting your fonts from the Google Fonts website will give you a new link that you can replace this one with.

Secondly, you will need to edit "style.css" at chapter 10 - FONTS. Now replace the names of the fonts with the new ones that you have chosen.

08. Scroll Reveal Effects

Table of Contents

When scrolling through a page on the site in your browser, you may have noticed elements fade into view with an animation effect. You can configure these by altering certain lines within the HTML files. Each element to be animated will be wrapped in a <div> and will look something like this:

<div class="wow fadeInDown" data-wow-duration="2s" data-wow-offset="250">
	...
</div>

This tells an element to fade in with a sliding down animation, will last two seconds, and will only start animating when the element is 250 pixels from the bottom of the browser window. Of course, you can change these to suit your needs or simply remove the line all together if you're not a fan! There are many animations to choose from and all of which can be found in the "animate.css" file.

10. The Gallery

Table of Contents

The gallery in this template uses a masonry layout which is handled by jQuery. To get the same grid effect seen in the demo, consult the image sizes found in the "images/demo/gallery" folder.

In order to have an image span across two columns, consider the following code as an example:

<div class="hungry-gallery-item wide wow fadeIn" data-wow-duration="2s" data-wow-offset="250">
	<a class="image-hover wide lightbox-gallery" href="images/demo/gallery/gallery-05.jpg">
		<img src="images/demo/gallery/gallery-05.jpg" alt="Gallery Image" />
		<div class="image-hover-overlay">
			<i class="fa fa-search-plus"></i>
		</div>
	</a>
</div>

This is the markup required for a gallery item and you will notice an extra CSS class called "wide" highlighted above. This will make the image span two columns provided it is big enough.

11. Menus

Table of Contents

Hungry uses a simple menu system that will link your menu items to either other pages within your site, or scroll to another section on the same page. The main navigation menu can be found near the top of each HTML file. Consider the following code with two different links;

<li><a href="index.html#hungry-about-us">About Us</a></li>
<li><a href="another-page.html">A Different Page</a></li>

The first menu item here will scroll the user to the "About Us" section on the same page, provided that the user is browsing the "index.html" file. Whereas the second menu item will direct the user to a different page.

Its important to note here that the href attribute needs the "index.html" part of the URL before the name of the section (which, here is "#hungry-about-us"). This is to ensure that should the user click on this link from another page, they will be directed to the correct HTML file and the correct section. The #hungry-about-us part of the URL is the ID of the section the user will get scrolled to.

13. Configuring the Reservation Form

Table of Contents

Hungry comes with a simple reservation form that your customers can fill out and send to you. It will give you information on that user including name, email, party number, phone number, date, time, and an optional message so they can ask questions. The form also has jQuery validation as well as back end validation. To configure the form, follow these simple steps...

Step 01

Make sure you have PHP installed on your web server. Most web hosts already have PHP installed, but if you are unsure, please contact your hosting provider. There are occasions where PHP might not be installed, such as on free web hosting.

Step 02

In the "php" folder, you will see a PHP file named "process-reservation.php". You will need to open this file with a text editor like notepad. Once you do, you will need to enter your email address and your name in the first to lines you see (you can't miss it, its commented!). This will ensure that everytime a user submits a form, it will get sent to your e-mail address.

Note: If you use a Gmail account, you will need to alter some more details further down the page, where it says "Gmail Configurations". Here you will also need to enter your email and password for your Gmail account. Everything else should be left untouched.

Step 03 (Optional)

All of the validation handled by jQuery can be found in "jquery-custom.js" which is found in the "js" folder. You can modify this if you need to display different error messages or translate them. The code can be found near the bottom of the Javascript file.

Step 04

Test it! Don't rely on your visitors to fill out your form all the time. You can also send some test bookings to ensure everything is working as it should be.

14. Frequently Asked Questions

Table of Contents
Q. I don't like the smooth scrolling! can I disable it please?

A. You can! edit "jquery-custom.js" in a text editor and remove the call to it if you prefer native browser scrolling. Or you can change its values. Here's the code you're looking for;

$('html').niceScroll({

	cursorcolor		: '#000',
	cursoropacitymin	: 0.4,
	cursoropacitymax	: 1,
	cursorborder		: '1px solid transparent',
	cursorborderradius	: '6px',
	zindex 			: 9999,
	scrollspeed		: 200

});
Q. What if I want to modify and use one of the jQuery scripts?

A. By default, each of the template files loads in the javascript file called "jquery-main.js". This file has all of the required scripts combined into this file and minified. This is to reduce the amount of requests and file size. If you need to modify any script, consider using a custom script or adding your own functions in "jquery-custom.js". If you do need to edit any scripts though, you will have to load each of the scripts directly. They can all be found in the "js" folder.
